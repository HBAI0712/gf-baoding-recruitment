# 上线前检查清单

这份清单给营业部负责人、综合岗、信息技术同事一起确认。逐项打勾即可。

## 一、内容确认

- [ ] 营业部名称无误。
- [ ] 联系人、电话、邮箱无误。
- [ ] 地址从“河北省”开始写，完整准确。
- [ ] 当前在招岗位准确。
- [ ] 如果没有在招岗位，首页显示“当前暂无在招岗位”。
- [ ] 福利待遇、补贴、休市假等表述已确认。
- [ ] 页面没有“高薪保证”“稳赚”“快速晋升”“资源变现”等承诺性表达。
- [ ] 页面不包含真实客户信息、账户信息、资产信息或未经授权的人像。

## 二、在线申请测试

- [ ] 打开正式网址，首页能正常访问。
- [ ] 点击“立即投递”，能进入在线申请页。
- [ ] 用测试信息提交一次申请。
- [ ] 上传 PDF 测试简历成功。
- [ ] 上传 DOC/DOCX 测试简历成功。
- [ ] 上传非简历格式文件会被拒绝。
- [ ] 超过大小限制的文件会被拒绝。
- [ ] 负责人邮箱能收到申请邮件。
- [ ] 邮件主题、候选人信息、简历附件都正确。

## 三、安全与部署确认

- [ ] 网站使用 HTTPS。
- [ ] `.env` 没有放到公开目录。
- [ ] `SMTP_PASS` 使用授权码，不是邮箱登录密码。
- [ ] `ALLOWED_ORIGIN` 已改成正式网站域名，不再使用 `*`。
- [ ] `MAIL_TO` 是招聘负责人邮箱。
- [ ] `RESUME_MAX_SIZE_MB` 已确认。
- [ ] `RATE_MAX` 和 `RATE_WINDOW_MS` 已确认，能限制异常高频提交。
- [ ] 后端健康检查 `/api/health` 返回正常。
- [ ] 非招聘网页来源提交会被拒绝。
- [ ] 服务器或网关已配置 HTTPS、访问日志和异常 IP 处置方式。
- [ ] 公司信息技术或服务商确认了服务器安全策略。

## 四、上线后维护

- [ ] 已保存一份原始压缩包备份。
- [ ] 已保存一份上线后的 `site-data.js` 备份。
- [ ] 负责修改内容的人知道如何打开 `editor.html`。
- [ ] 负责修改内容的人已阅读 `傻瓜式编辑说明.md`。
- [ ] 负责修改内容的人知道只需要替换 `site-data.js`，一般不用改代码。
