# 公网部署交接单

目标网址建议使用带 `gf` 和 `baoding` 的名称，例如：

```text
https://gf-baoding-recruitment.onrender.com
https://guangfa-baoding-recruitment.onrender.com
https://gf-baoding-jobs.onrender.com
```

如果使用公司正式域名，可考虑：

```text
https://gf-baoding-jobs.公司域名
https://jobs-baoding.公司域名
```

## 一、已准备好的部署文件

网站目录：

```text
recruitment-site
```

里面已经包含：

- `index.html`：招聘首页
- `apply.html`：在线申请页
- `editor.html`：傻瓜式内容编辑器
- `site-data.js`：可编辑内容数据
- `server/server.js`：在线申请与邮件发送后端
- `Dockerfile`：容器部署文件
- `docker-compose.yml`：自有服务器部署文件
- `render.yaml`：Render 一键部署蓝图
- `server/.env.example`：环境变量模板
- `公网部署说明.md`：技术上线说明
- `傻瓜式编辑说明.md`：非技术人员编辑说明
- `上线前检查清单.md`：上线验收清单

## 二、真正上线还需要补齐的信息

这些信息不能凭本机权限自动生成，需要公司或账号持有人提供：

- Render、Railway、Vercel、云服务器或公司服务器账号权限。
- 正式公网域名，或允许使用平台分配域名。
- 发件邮箱 SMTP 授权码。
- 是否允许使用 `380704292@qq.com` 接收所有简历。
- 公司信息安全、合规和品牌审核确认。

## 三、推荐上线方式

优先推荐 Render 或公司服务器 Docker 部署，因为它可以同时托管网页和后端接口。

### Render 推荐配置

服务名：

```text
gf-baoding-recruitment
```

运行方式：

```text
Docker
```

健康检查：

```text
/api/health
```

必须填写的环境变量：

```text
ALLOWED_ORIGIN=https://gf-baoding-recruitment.onrender.com
SMTP_HOST=smtp.qq.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=发件邮箱@qq.com
SMTP_PASS=SMTP授权码
MAIL_TO=380704292@qq.com
TRUST_PROXY=true
```

如果 Render 最终分配的网址不是 `gf-baoding-recruitment.onrender.com`，请把 `ALLOWED_ORIGIN` 改成实际网址。

## 四、防垃圾和安全状态

当前版本已经包含：

- 表单隐藏反垃圾字段。
- 同一来源提交频率限制。
- 只允许 PDF、DOC、DOCX 简历。
- 后端检查简历文件头，降低伪装文件风险。
- 限制简历大小。
- 限制正式来源域名。
- 基础安全响应头。
- 隐藏后端源码目录。

仍建议正式部署时配置：

- HTTPS。
- 平台防火墙或 WAF。
- 服务器访问日志。
- SMTP 授权码定期更换。
- 简历邮件处理和删除制度。

## 五、上线后编辑方式

普通同事不需要登录服务器。

1. 打开 `editor.html`。
2. 改文字。
3. 下载新的 `site-data.js`。
4. 交给网站维护人员替换服务器上的同名文件。
5. 刷新正式网址检查。
