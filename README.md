# 广发证券保定营业部招聘动态网页使用说明

这是一套可长期使用的招聘网页文件。以后再招聘时，可以自己修改岗位、福利、联系人、地址等内容。

## 1. 每个文件是做什么的

- `index.html`：正式招聘网页，给应聘者看的。
- `apply.html`：在线咨询与投递页面，候选人在这里填写称谓、联系电话并上传简历。
- `editor.html`：内容编辑器，供营业部后续自行修改内容使用。
- `site-data.js`：网页内容文件，编辑器下载的新文件就是它。
- `styles.css`：网页外观文件，一般不用改。
- `app.js`：网页动态效果文件，一般不用改。
- `apply.js`：申请表单提交脚本，一般不用改。
- `assets/baoding-cityscape.svg`：原创保定城市线稿背景，无照片版权风险。
- `assets/baoding-cityscape.svg`：原创保定城市线稿，不使用网上照片，避免照片版权问题。
- `server` 文件夹：后端邮件发送模板。部署后，申请表单才能把信息和简历自动发送到负责人邮箱。
- `傻瓜式编辑说明.md`：给没有技术基础的同事看的日常编辑说明。
- `公网部署说明.md`：给公司技术、信息科技或第三方服务商看的上线说明。

## 2. 如何预览网页

1. 打开 `recruitment-site` 文件夹。
2. 双击 `index.html`。
3. 浏览器会打开招聘网页。

手机预览需要先把网页上传到一个网址，或者让技术同事放到公司服务器/内网服务器上。

## 3. 如何修改招聘内容

推荐用 `editor.html`，不用懂代码。

1. 打开 `recruitment-site` 文件夹。
2. 双击 `editor.html`。
3. 在页面里修改内容，例如：
   - 首页标题和介绍
   - 联系人、电话、邮箱、地址
   - 招聘要求
   - 福利待遇
   - 岗位名称、岗位标签、岗位简介、岗位职责
   - 合规提示
4. 改完后，点击页面上方的“1 保存并下载内容文件”。
5. 浏览器会下载一个新的 `site-data.js` 文件。
6. 用这个新下载的 `site-data.js` 替换原文件夹里的旧 `site-data.js`。
7. 再双击打开 `index.html`，刷新后就能看到新内容。

如果只负责日常改字，请优先看：

```text
傻瓜式编辑说明.md
```

注意：Logo 当前使用广发证券官网 CDN 的矢量 logo，不在编辑器里修改。若以后公司统一下发新的品牌素材，请让技术人员在 `index.html` 和 `apply.html` 中替换 logo 地址。

## 4. 修改岗位时怎么填

在编辑器里，每个岗位都有 4 个部分：

- 岗位名称：例如 `财富管理方向`
- 岗位标签：例如 `重点招募`
- 岗位简介：一句话说明这个岗位适合什么样的人
- 岗位职责：一行写一条，例如：

```text
学习金融市场、证券业务和客户服务流程
协助开展客户沟通、需求理解和日常服务
参与投资者教育、活动组织和客户陪伴
```

如果要增加岗位，点击“新增岗位”。  
如果要减少岗位，点击“删除最后一个岗位”。

如果当前没有招聘岗位，可以一直点击“删除最后一个岗位”，直到岗位列表为空。  
保存后，招聘首页会自动显示“当前暂无在招岗位”，申请页也不会开放投递。

## 5. 修改福利待遇时怎么填

福利待遇是一行一条，例如：

```text
五险二金
企业年金
优才补贴
入职补贴
工会福利
定期体检
节日福利
行业独享休市假（无调休）
```

建议福利、补贴、休假等内容发布前先让公司合规或人事确认。

## 7. 如何上线到各平台都能访问

如果只需要展示招聘网页，技术同事上传整个 `recruitment-site` 文件夹即可。

可以放到：

- 公司官网
- 公司或营业部服务器
- 公众号菜单外链
- H5 页面托管平台
- 对象存储静态网站服务

上线时一定要保持文件结构不变。也就是说，`index.html`、`apply.html`、`site-data.js`、`styles.css`、`app.js`、`apply.js`、`assets` 文件夹都要一起上传。

如果需要所有人都可以访问并在线申请，请优先阅读：

```text
公网部署说明.md
```

正式招聘建议使用公司服务器、公司官网目录或公司认可的云服务，不建议用个人电脑临时对外提供访问。

当前网站包已准备好公网部署所需的基础文件，包括 `server` 后端服务、`Dockerfile`、`docker-compose.yml`、`.env.example`、安全与防垃圾提交配置、上线前检查清单。

## 8. 如何让申请表单自动发邮件

只上传静态网页，不能自动发送简历附件到邮箱。  
要实现“填写信息 + 上传简历 + 自动发到负责人邮箱”，必须部署 `server` 文件夹里的后端服务。

给技术同事看：

1. 打开 `server/README.md`。
2. 按说明安装 Node.js 依赖。
3. 配置发件邮箱的 SMTP 授权码。
4. 启动后端服务。
5. 确认 `site-data.js` 里的 `applyEndpoint` 指向后端接口。

当前默认接口是：

```text
/api/applications
```

如果前端和后端部署在同一个域名下，一般不用改。  
如果后端是单独域名，需要改成完整地址，例如：

```text
https://api.example.com/api/applications
```

上线后可以访问下面这个地址检查后端是否正常：

```text
https://你的域名/api/health
```

## 10. 合规提醒

发布前建议公司合规和品牌部门审核：

- 福利、补贴、岗位、录用条件以公司制度和实际录用通知为准。
- 避免使用“高薪”“稳赚”“保证成长”“快速晋升”“资源变现”等承诺性表达。
- 避免展示真实客户信息、账户信息、资产信息或未经授权的人像。
- 页面仅用于招聘信息展示，不构成投资建议或收益承诺。

## 11. 常见问题

**改完内容为什么网页没变化？**  
通常是没有替换旧的 `site-data.js`，或者浏览器缓存了旧内容。请确认替换成功，然后刷新网页。

**能不能以后继续改？**  
可以。每次打开 `editor.html`，修改后下载新的 `site-data.js`，再替换旧文件即可。

**能不能做成真正后台登录编辑？**  
可以，但那需要服务器和后台系统。这一版是更稳定、更容易部署的静态动态网页，适合招聘页长期保存和快速修改。

**为什么我本地点击提交申请，提示不能发邮件？**  
因为本地双击打开网页只是预览，没有后端服务。把 `server` 后端部署好以后，表单才能真正发送邮件和附件。
