# 广发证券保定营业部招聘网站公网部署说明

目标：让所有人可以通过一个网址访问招聘网页，并且候选人可以在线填写信息、上传简历，系统自动发送到招聘负责人邮箱。

## 1. 必须具备的三样东西

1. 一个公网网址  
   例如 `https://baoding.gf.com/jobs` 或 `https://jobs.example.com`

2. 一个可以运行 Node.js 的服务器  
   用来接收申请表单、处理简历附件、发送邮件。

3. 一个发件邮箱 SMTP 授权码  
   例如 QQ 邮箱、企业邮箱等。注意：这里要用“SMTP 授权码”，不要把邮箱登录密码交给网页。

## 2. 推荐上线方式

### 方案 A：前端和后端部署在同一个服务器、同一个域名下

这是最推荐的方式，最稳定，也最少改文件。

部署后访问：

```text
https://你的域名/
```

在线申请接口保持默认：

```text
/api/applications
```

也就是说，`site-data.js` 里的这一行不用改：

```js
applyEndpoint: "/api/applications",
```

### 方案 B：网页和后端分开部署

如果网页放在静态网站平台，后端单独放在另一台服务器，需要把 `site-data.js` 改成后端完整接口地址：

```js
applyEndpoint: "https://你的后端域名/api/applications",
```

同时，后端 `.env` 里的 `ALLOWED_ORIGIN` 要改成网页正式域名：

```text
ALLOWED_ORIGIN=https://你的网页域名
```

### 方案 C：Render 一键部署

当前目录已准备好 `render.yaml`，服务名为：

```text
gf-baoding-recruitment
```

如果这个名称可用，Render 分配的网址通常会类似：

```text
https://gf-baoding-recruitment.onrender.com
```

Render 创建服务后，需要在环境变量里补齐 SMTP 授权码，并把 `ALLOWED_ORIGIN` 改成最终分配的网址。

## 3. 给技术同事的部署步骤

### 方式一：普通 Node.js 部署

进入后端目录：

```bash
cd server
```

安装依赖：

```bash
npm install
```

复制环境变量模板：

```bash
cp .env.example .env
```

编辑 `.env`：

```text
PORT=3000
ALLOWED_ORIGIN=https://你的正式网站域名

SMTP_HOST=smtp.qq.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=发件邮箱@qq.com
SMTP_PASS=SMTP授权码

MAIL_TO=380704292@qq.com
RESUME_MAX_SIZE_MB=12
RATE_WINDOW_MS=900000
RATE_MAX=20
```

启动服务：

```bash
npm start
```

### 方式二：Docker 部署

如果服务器已安装 Docker，可以在 `recruitment-site` 目录下执行：

```bash
cp server/.env.example server/.env
```

先把 `server/.env` 按正式域名和邮箱授权码填写完整，然后执行：

```bash
docker compose up -d --build
```

服务默认监听：

```text
http://服务器地址:3000
```

正式公网访问建议再由 Nginx、公司网关或云服务反向代理到 HTTPS 域名。

健康检查地址：

```text
https://你的域名/api/health
```

如果返回：

```json
{ "ok": true, "service": "gf-baoding-recruitment" }
```

说明后端服务已经运行。

## 4. 正式上线前必须测试

1. 打开正式网址，确认首页可以访问。
2. 点击“立即投递”，进入在线申请页。
3. 填写测试姓名、手机号、岗位方向。
4. 上传一个测试 PDF 或 Word 简历。
5. 点击提交。
6. 检查 `380704292@qq.com` 是否收到邮件。
7. 确认邮件里包含：
   - 称谓 / 姓名
   - 联系电话
   - 应聘方向
   - 当前状态
   - 补充说明
   - 简历附件

测试完成后，删除测试邮件和测试简历。

## 5. 域名、备案和合规提醒

如果网站部署在中国大陆服务器并使用正式域名，通常需要由公司或云服务商协助完成网站备案等手续。  
如公司已有官网域名，推荐优先放在公司官网或公司已备案域名下。

上线前建议由公司相关部门确认：

- 域名和服务器是否符合公司信息安全要求。
- 招聘信息、福利待遇、岗位要求是否经过合规或人事审核。
- 简历信息和附件的接收、保存、删除流程是否符合公司个人信息保护要求。
- 页面是否可以对外公开使用广发证券 logo 和营业部名称。

## 6. 防垃圾、防攻击配置

当前网站包已做好的基础防护：

- 申请接口限制提交频率，默认同一来源 15 分钟最多 20 次。
- 只允许 PDF、DOC、DOCX 简历格式。
- 后端会检查文件内容开头，避免把伪装成简历的文件直接转发到邮箱。
- 简历大小默认限制为 12MB。
- 表单有隐藏反垃圾字段，普通用户看不到，机器乱填会被静默处理。
- 生产环境可限制 `ALLOWED_ORIGIN`，只接受正式招聘网址提交。
- 后端隐藏技术栈标识，并设置基础安全响应头。
- `/server` 目录不会被公开访问。

正式上线建议额外配置：

- 网站必须使用 HTTPS。
- `ALLOWED_ORIGIN` 必须改成正式招聘网址，不建议公网长期使用 `*`。
- 服务器放在公司认可的网关、WAF、云防火墙或反向代理后面。
- 根据实际访问量调整 `RATE_MAX`，异常提交明显增多时可临时调低。
- 定期查看服务器日志，发现异常 IP 可由技术同事在网关层封禁。
- `.env`、SMTP 授权码和服务器登录信息不得发给无关人员。

## 7. 上线后如何继续编辑招聘内容

内容仍然通过 `editor.html` 修改。

修改后下载新的 `site-data.js`，替换服务器上的旧文件即可。  
如果只修改岗位、福利、联系人、地址，一般不需要重新部署后端。

不会技术的同事请看：

```text
傻瓜式编辑说明.md
```

## 8. 常见问题

### 只把网页上传到静态平台，能不能在线收简历？

不能。静态网页只能展示内容，不能安全地发送简历附件到邮箱。  
在线申请必须部署 `server` 后端。

### 能不能直接用个人电脑当服务器？

不建议。个人电脑关机、断网、换网络后，外部人员就访问不了。正式招聘建议使用公司服务器或云服务器。

### 可以放到公众号菜单里吗？

可以。公众号菜单本质上是打开一个网址。前提是网站已经有公网 HTTPS 地址。

### 可以生成二维码吗？

可以。网站正式上线后，把正式网址生成二维码即可，用于海报、朋友圈、公众号文章或短视频主页。
